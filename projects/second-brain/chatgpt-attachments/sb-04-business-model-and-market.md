# Second Brain — Business Model & Market

## Beachhead Market Strategy

### Why Start With University Project Teams

The fastest path to validating the core thesis is a user group that:
- Has acute knowledge-handoff pain (team projects with rotating members)
- Is accessible at zero acquisition cost
- Has a natural network effect trigger (invite teammates)
- Makes decisions fast (students, not enterprise procurement)

**Phase 1: University project / capstone teams**
- Pain: teams forget decisions mid-project, handoff between teammates is lossy, knowledge disappears when the project ends
- Target: 3-5 teams = 15-30 users for initial validation
- Timeline: pilot May-July 2026
- Cost to acquire: ~$0

**Phase 2: University research labs**
- Pain: graduate students carry institutional knowledge that evaporates when they graduate
- Pain: professors repeat the same onboarding every 2 years with new grad students
- Access path: referral from pilot teams

**Phase 3: Thai startups (10-50 people)**
- High growth = high onboarding pain = strongest PMF signal
- Turnover problem = Institutional Memory pitch lands hardest here

**Long-term market:**
Thai SMEs + organizations with high turnover (agencies, consulting, government projects)

- Pro at ~$20 benchmarks against ChatGPT Plus / Claude Pro (same tier, different positioning)
- Starter at ฿199 creates an entry point for students and individuals wanting to try before committing

## Validation Framework

### Primary Metric: Day-3 Retention (Unprompted Return)

The only signal that matters for PMF at this stage: do users come back on Day 3 without being prompted?

| Signal | Threshold | Action |
|--------|-----------|--------|
| 🟢 PMF (GO) | ≥ 3/5 users return Day 3 unprompted | Proceed to A2A collab layer |
| 🟡 Conditional | 2/5 return | Fix onboarding first, don't pivot yet |
| 🔴 Pivot signal | 0-1/5 return | Job-to-be-done is wrong; change target user |

### Secondary Signals to Watch

- Do users invite a teammate? (network effect signal)
- Do users reference something the AI remembered from a previous session? (memory value signal)
- Do any users try to use it for team handoff without being prompted? (A2A demand signal)
- Does anyone pay without asking? (willingness-to-pay signal)

### Network Effect Warning

The AI-to-AI value is lagged — users need to onboard individually before the network benefits appear. Don't rely on viral growth from the pilot. Hand-select and onboard each user manually. Measure solo value first.

## Why This Business Model Is Hard to Copy

1. **Data moat**: Each user's brain deepens over time. Switching means starting from zero. After 3-6 months of use, the switching cost is a person's entire decision history.

2. **Network effect**: The more people in an org use it, the more valuable the AI Network becomes. Orgs can't get this value by having individuals use separate tools.

3. **Protocol lock-in (positive)**: Once org AI-to-AI workflows are built on A2A, switching breaks the workflows — not just the individual experience.

4. **Model-agnostic positioning**: We never compete with LLM providers. As better models come out, our product gets better without us doing anything. We're the memory + network layer, not the model.
