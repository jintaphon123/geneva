# Cross-Handoff Generator

You are generating a handoff file that will be read by **another person's AI**, not by a human.
Write it so an AI can immediately understand context and help the next person continue without any briefing.

## Step 1: Gather context

Ask the user (if not already clear from the session):
- What project / task is this handoff for?
- Who is receiving this? (name or role)
- What was the goal this session?

## Step 2: Generate the handoff file

Output a markdown block with this exact structure:

```markdown
# Geneva Handoff — {date} | {project_name}

**From:** {sender_name or "teammate"}  
**To:** {receiver_name or "next person"}  
**Session goal:** {one sentence}

---

## What was done

{3-5 bullets of concrete actions taken}

---

## Key decisions (with WHY)

| Decision | Why we chose this | What we rejected |
|----------|------------------|-----------------|
| {decision 1} | {reason — the "why" behind it} | {alternative} |
| {decision 2} | {reason} | {alternative} |

---

## Context the next person must know

{2-4 short paragraphs of non-obvious context that isn't in the files}
- Any constraints, edge cases, or gotchas discovered
- Any agreements or assumptions made
- Any external dependencies or waiting items

---

## Open items / blockers

- [ ] {open item 1}
- [ ] {open item 2}
- ⛔ {blocker — what's blocking and who needs to unblock it}

---

## Suggested first question to ask your AI

> "{a specific question the next person should ask their AI to pick up where this left off}"

---

## Files to load

{list of files the next person's AI should read first}
- `{path/to/file}` — {why}

---
_Generated by Geneva cross-handoff • {timestamp}_
```

## Step 3: Save and share

Tell the user:
1. Save this as `handoff-{date}-{project}.md` in your shared project folder
2. Send the file to your teammate via LINE / Discord / however your team communicates
3. Tell them: "รัน `/receive-handoff` ใน Geneva แล้วแนบไฟล์นี้"
